Why Plu-Stan

Ship safer Plinth code

Automated detection of common pitfalls specific to on-chain Cardano code — security flaws, performance issues, and unsafe patterns.

🛡️

Security Rules

Detect missing signature verification invariants, unsafe credential comparisons, unvalidated deserialization, currencySymbolValueOf misuse, and validity interval pitfalls.

⚡

Performance Rules

Flag expensive on-chain patterns: higher-order list helpers, nested list traversals, non-strict bindings, guard syntax, optional types, and division-before-multiplication precision loss.

🔌

VS Code / Cursor Extension

Standalone extension with explorer tree view, per-module or workspace analysis, and diagnostics in the Problems panel.

🧬

Built on Stan

Extends the Stan static analysis framework with Plinth-specific rules for on-chain Cardano code, inheriting all existing Stan inspections.

🖥️

CLI with JSON Output

The plustan CLI supports machine-readable JSON output for editor integrations, CI pipelines, and targeted analysis of individual on-chain modules.

Detection Rules

What Plu-Stan catches

13 automated detection rules covering security vulnerabilities and performance anti-patterns specific to on-chain Plinth code — on top of all inherited Stan rules.

Security & Correctness Warnings
PLU-STAN-01 · Signature Verification Builtin Invariants
Using verifyEd25519Signature, verifyEcdsaSecp256k1Signature, or verifySchnorrSecp256k1Signature requires direct on-chain verification of message hash correspondence and replay prevention.
Warning
PLU-STAN-03 · Optional Types in On-Chain Code
Using Maybe or Either types is an anti-pattern; prefer fast-fail variants or continuation functions.
Warning
PLU-STAN-04 · Equality on PubKeyHash / ScriptHash / Credential
Comparing only credentials instead of full Address can lead to staking value theft.
Warning
PLU-STAN-09 · valueOf in Equality Comparisons
Using valueOf with adaSymbol and adaToken in comparisons can be unsafe if the token set is unbounded.
Warning
PLU-STAN-10 · Unvalidated Hashes from BuiltinData
Comparing Address/ScriptHash/PubKeyHash/Credential from unsafeFromBuiltinData without validating ledger invariants can create unsatisfiable constraints.
Warning
PLU-STAN-11 · Usage of currencySymbolValueOf
Does not enforce that all token amounts are strictly positive or negative; allows mixed mint/burn in the same transaction.
Warning
PLU-STAN-12 · Validity Interval / POSIX Time Misuse
Using validity interval utilities or txInfoValidRange without ensuring finite bounds can lead to unbounded time windows.
Warning
PLU-STAN-16 · Division Before Multiplication
Division before multiplication in integer arithmetic causes precision loss due to rounding; multiply first, then divide.
Warning
Performance
PLU-STAN-02 · Usage of unsafeFromBuiltinData
Using unsafeFromBuiltinData without integrity checks can lead to unbounded datum spam attacks.
Performance
PLU-STAN-05 · Higher-Order List Helpers
Functions like filter, any, all, find are inefficient in on-chain code; prefer specialized recursive functions.
Performance
PLU-STAN-06 · Multiple List Traversals
Nested list operations (e.g., map over filter) cause redundant iterations and increase execution costs.
Performance
PLU-STAN-07 · Guard Syntax in On-Chain Code
Guard syntax produces inefficient UPLC; prefer if-then-else or lower-level conditionals.
Performance
PLU-STAN-08 · Non-Strict Let Binding Used Multiple Times
Non-strict bindings used multiple times cause repeated evaluation; make bindings strict with bang patterns.
Performance
