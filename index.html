What Plu-Stan catches

A growing set of automated rules covering security vulnerabilities and performance anti-patterns specific to on-chain Plinth code.

Missing Signature Verification
Cryptographic signature functions without on-chain hash verification and replay prevention.
Security
Partial Credential Comparison
Comparing only credentials instead of full addresses can allow staking value theft.
Security
Unsafe valueOf Usage
Using valueOf with adaSymbol/adaToken in comparisons when the token set is unbounded.
Security
Unvalidated Deserialization
Comparing values from unsafeFromBuiltinData without validating ledger invariants.
Security
Higher-Order List Functions
Functions like filter, any, all, find are inefficient on-chain — prefer hand-rolled recursion.
Performance
Nested List Operations
Chaining map over filter causes redundant iterations and inflated execution costs.
Performance
Non-Strict Bindings
Lazy bindings used multiple times cause repeated evaluation — use bang patterns.
Performance
